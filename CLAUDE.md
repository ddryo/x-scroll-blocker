## 言語

日本語でレスポンスしてください。

## 動作ルール

実装・設計・ファイル変更を伴う作業を勝手に行わないこと。
ユーザーが `/project-autopilot` などのタスク実装コマンドを提示するか、または明示的に実装許可を出すまで待って下さい。
質問への回答や簡単な説明は即座に対応して構いません。

### 重要

Chrome拡張機能の作り方・関連コードに関しては常に最新情報をチェックすること。

## 記述ルール

スキルやカスタムコマンド、markdownドキュメントで使用する共通の記法：

| 記法           | 意味                       | 例                     |
| -------------- | -------------------------- | ---------------------- |
| `{...}`        | 置き換えるプレースホルダー | `{プロジェクト名}`     |
| `[...]`        | 任意・オプションの入力     | `[ユーザーからの指示]` |
| `<!-- ... -->` | 記入ガイド                 | `<!-- 1文で簡潔に -->` |
| `...`          | 必要に応じて行を追加       | 表の追加行など         |

## プロジェクトのドキュメント

- プロジェクトの仕様書: `.docs/spec.md`
- 実装ロードマップ: `.docs/roadmap.md`

## ワークフロー

要件定義作成からの初回実装:

```
/project-init
    ↓
/project-planning [要望 or TODO.mdを読み取り]
    ↓
/project-autopilot

```

機能追加/バグフィックスを行う時:

```
/project-new-task [要望]
    ↓
/project-autopilot --start=T-M{X}-{N}
```

全体を見直したい場合:

```
/project-replan [追加要望]
    ↓
/project-autopilot
```
